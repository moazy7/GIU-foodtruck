# Screenshots Guide

Add screenshots of all implemented pages to this directory. Below is a recommended list of pages to capture for documentation:

## Customer Pages
- **customerHomepage.png** - Customer home page with truck listings
- **trucks.png** - Trucks list view
- **truckMenu.png** - Truck menu view (customer browsing)
- **menuItems.png** - Menu items view with categories
- **cart.png** - Shopping cart with items and total
- **myOrders.png** - Customer's order history
- **orderDetails.png** - Details of a specific order

## Truck Owner Pages
- **ownerDashboard.png** - Owner dashboard overview
- **truckOwnerHomePage.png** - Truck owner home page
- **truckOrders.png** - Incoming orders for truck owner
- **addMenuItem.png** - Form to add new menu item
- **truckMenu_owner.png** - Truck menu view (owner management)
- **menuItemEdit.png** - Edit menu item form

## Authentication Pages
- **register.png** - User registration page
- **login.png** - User login page

## Instructions
1. Navigate through the application and capture screenshots of each page
2. Save them as PNG files in this directory with clear, descriptive names
3. Reference them in the main README.md using markdown image syntax:
   ```
   ![Page Name](public/images/screenshots/pageName.png)
   ```
4. Commit the screenshots to the repository

## Naming Convention
Use camelCase or kebab-case with the .png extension. Examples:
- `customerHomepage.png`
- `truck-menu.png`
- `add-menu-item.png`
